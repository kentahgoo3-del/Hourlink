🔍 Why this is happening
1. 🌐 Device language / locale issue (MOST COMMON)

Your phone might be set to:

Chinese (Simplified / Traditional), OR
English + Chinese as secondary language

👉 Apps sometimes auto-switch parts of UI based on system locale.

2. 🧠 Default localization fallback

Your app likely:

Has Chinese as default language
Missing English translations for some keys → falls back to Chinese

That’s why only certain buttons show Chinese:

“New Timer” → correct (English exists)
Icons/buttons → Chinese (no English mapping)
3. 🧩 Hardcoded text in the app

Some text may literally be coded like:

text: "切"

instead of using translation keys.

🛠️ How to fix it (step-by-step)
✅ Fix #1: Force English on your phone

On Android:

Go to Settings
Go to General Management
Tap Language
Make sure:
English is first
Remove Chinese if present
✅ Fix #2: Force English inside your app (best solution)

If you're building this app (which you are 💡), do this:

React Native example:
import * as Localization from 'expo-localization';
import i18n from 'i18n-js';

i18n.locale = 'en'; // FORCE ENGLISH
i18n.fallbacks = true;
